Introduction

@RestTickets are simple, customizable tables that store various types of information. This website was created to provide a REST API for managing these individual tickets using the HTTP protocol. This website also gives some tools that help both visualize and conceptualize the provided REST services.

API

This API will only accept requests if they are from a valid user. You can either make the requests using the tools provided by the site, or you can use your own client to make the requests. If you make the HTTP requests from your own client, be sure to include your authorization header.

What do I put in the authorization header?

The authorization header takes a Base64 encoded combination of your username and password (the same ones you used to register). The combination is of the format "username:password", with a colon in between. An example authorization header looks like this: 

Authorization: Basic QWxhZGRpbjpvcGVuIHNlc2FtZQ==
This is necessary in order to authenticate the requests and prohibit restricted users from interacting with the services.

What about the content-type?

This service requires that information sent from your client's request body must be in json format. An example of the json header looks like this: 

Content-Type: application/json
More details outlining the actual json format of the body will be described in the following sections.

How are the resources structured?

Each User has the ability to manage both Projects and Tickets. Furthermore, a User can also add other users to their own project(s), but Users can never add themselves to another User's project(s). So if we were to create a simple model for our application structure, it would be in a linear fashion: 

Users --> Projects --> Tickets
Users contain projects, and projects contain tickets.

What are all the URL endpoints?

Keep reading below. All of the REST operations can be found here on this page.

How can I get started?

You can either start using your own client to make API requests, or you can use the tools provided on this website if you prefer to use a GUI.

@get

URL
Description
/projects/

Get all of the projects associated with your user account (see above about auth headers).

/projects/{project_id}

Get details of a single project using the project's p_id (project id).

/projects/{project_id}/tickets

Get all of the tickets associated with a specific project.

/projects/{project_id}/tickets/{ticket_id}

Get details of a single ticket associated with a specific project.

/projects/{project_id}/users/

Get the usernames of every user associated with a specific project.

@post

URL
Request Body
Description
/projects/
{"project_name" : "name",
"project_description" : "description",
"project_category" : "category"}

Post a single project.

/projects/{project_id}/tickets
{"ticket_name" : "name",
"ticket_description" : "description"}

Post a single ticket for a specific project.

/projects/{project_id}/users
{"username" : "user"}

Post a single user to a specific project.

@put

URL
Request Body
Description
/projects/{project_id}
{"project_name" : "updated name",
"project_description" : "updated description",
"project_category" : "updated category"}

Update a single project's fields by specifying the fields you want to update.

/projects/{project_id}/tickets/{ticket_id}
{"ticket_name" : "name",
"ticket_description" : "description"}

Update a single ticket's fields by specifying the fields you want to update.

@delete

Warning: these operations are irreversible!

URL
Description
/projects/

Delete all of the projects associated with your user account.

/projects/{project_id}

Delete a single project using the project's p_id (project id).

/projects/{project_id}/tickets

Delete all of the tickets associated with a specific project. 

/projects/{project_id}/tickets/{ticket_id}

Delete a single ticket associated with a specific project.

/projects/{project_id}/users

Delete all of the users (other than the user executing this command) associated with a specific project.

/projects/{project_id}/users/{username}

Delete a specific user (by username) from this specific project.
