About

This website is designed to help users understand the fundamentals of RESTful architectural design by providing a minimal REST API. The REST Tickets themselves can be thought of as just rows in a MySQL table. Each ticket property is defined by a separate column, and each user has control of their own tickets. Tickets are modeled under projects, and projects are modeled under users. Users can create and modify both projects and tickets.

How do I use this?

To use the API, you can either use your own client, or the built-in tools that come with this website. All of the responses are in JSON format.

What did you use to make this website?

This web app was built using Flask, a microframework for Python. The Flask app runs on top of Tornado server, which implements non-blocking network I/O.

How can I make an API like this?

All of this code is open sourced and available on GitHub at https://github.com/recio862/RestTickets.
